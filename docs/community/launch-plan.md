# AKCP Orchestrator Launch Plan

## The Problem Narrative

AI Agents are fast, but their Context Windows are full of noise. Projects like OpenWiki proved that agents need curated, structured documentation. However, plain Markdown isn't enough for enterprise deployments. Enterprises need **Agent-Ready Knowledge**: structured data (OKF) delivered through safe, governed, token-budgeted pipes (MCP).

## Launch Objectives

1. **Educate:** Teach developers what "Agent-Ready Knowledge" means.
2. **Onboard:** Get 100 developers to run the 5-Minute Quickstart (no LLM required).
3. **Extend:** Encourage community creation of new Domain Adapters and MCP Capabilities.

## Core Messaging

- **For Developers:** "Stop writing mega-prompts. Manage your agent's memory like you manage your codebase."
- **For Enterprises:** "Unleash AI agents safely. Control exactly what they can read, execute, and submit."

## Launch Phases

### Phase 1: The "Show, Don't Tell" Release (Current)

- Release the 5-Minute Quickstart (Terminal based, no API keys).
- Publish the comparison matrix showing why RAG and raw MCP aren't enough.
- Call to Action: _Try the CLI and generate your first Context Pack._

### Phase 2: Domain Expansion

- Announce Domain Adapters (`software-project`, `it-operations`, `compliance`).
- Host a community bounty for the best custom `Domain Adapter` or `Eval Scenario`.
- Call to Action: _Submit a PR for a new Domain Adapter._

### Phase 3: Enterprise Trust

- Publish the NIST AI RMF and OWASP mapping whitepapers.
- Highlight the `Agent-Ready Control Plane` capability.
- Call to Action: _Adopt AKCP for your internal AI platform._
