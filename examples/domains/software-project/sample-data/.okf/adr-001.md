# Use SQLite for Approval Store

## Context

We need a persistent store for pending HITL approval tokens that survives server restarts.

## Decision

We will use SQLite due to its zero-configuration setup, reliability, and fast file-based reads, avoiding the operational overhead of Redis for single-node deployments.

## Consequences

- Requires a persistent volume if deployed on Kubernetes.
- Simplifies local development drastically.
